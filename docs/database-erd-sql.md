# Database ERD/SQL

The task management service uses a relational schema with three main tables.

## ERD Summary

### `app_user`
Stores user credentials, role, activation status, and audit metadata.

Key fields:
- `id` primary key
- `username` unique
- `password`
- `role`
- `active`
- `created_at`, `updated_at`, `created_by`, `updated_by`

### `task`
Stores task records and ownership information.

Key fields:
- `id` primary key
- `title`
- `description`
- `status`
- `owner_id` foreign key to `app_user(id)`
- `deleted` soft delete flag
- `created_at`, `updated_at`, `created_by`, `updated_by`

### `task_comment`
Stores comments attached to tasks.

Key fields:
- `id` primary key
- `task_id` foreign key to `task(id)`
- `author_id` foreign key to `app_user(id)`
- `content`
- `created_at`, `updated_at`, `created_by`, `updated_by`

## Relationships
- One user can own many tasks
- One task can have many comments
- One user can author many comments

## SQL Schema
```sql
CREATE TABLE app_user (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    role VARCHAR(20) NOT NULL,
    active BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by VARCHAR(100) NOT NULL,
    updated_by VARCHAR(100) NOT NULL
);

CREATE TABLE task (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title VARCHAR(150) NOT NULL,
    description VARCHAR(1000) NOT NULL,
    status VARCHAR(30) NOT NULL DEFAULT 'PENDING',
    owner_id BIGINT NOT NULL,
    deleted BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by VARCHAR(100) NOT NULL,
    updated_by VARCHAR(100) NOT NULL,
    CONSTRAINT fk_task_owner FOREIGN KEY (owner_id) REFERENCES app_user(id)
);

CREATE TABLE task_comment (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    task_id BIGINT NOT NULL,
    author_id BIGINT NOT NULL,
    content VARCHAR(500) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by VARCHAR(100) NOT NULL,
    updated_by VARCHAR(100) NOT NULL,
    CONSTRAINT fk_comment_task FOREIGN KEY (task_id) REFERENCES task(id),
    CONSTRAINT fk_comment_author FOREIGN KEY (author_id) REFERENCES app_user(id)
);

CREATE INDEX idx_task_owner ON task(owner_id);
CREATE INDEX idx_task_status ON task(status);
CREATE INDEX idx_task_deleted ON task(deleted);
CREATE INDEX idx_comment_task ON task_comment(task_id);
CREATE INDEX idx_comment_author ON task_comment(author_id);
```

## Seed Data
The database migration inserts a default admin account if it does not already exist:
- Username: `admin`
- Role: `ROLE_ADMIN`
